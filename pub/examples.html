An Ahnentafel ("ancestor table") listing a person's direct ancestors

The most basic family tree, in the structure of an inverse binary tree. 


const greeAhnentafel = new PedigreeTree("ahnentafel");
const demoAhnentafel = document.getElementById('demoAhnentafel');
greeAhnentafel.setGraph({height: "520px"});
greeAhnentafel.generate(demoAhnentafel, ahnentafel);

Note that other than limiting the height of the graph to 520px, the options are left as their defaults.
We can see that the graph contents have scaled to fit the height of 520px.
Also note that if an additional detail pop-up is open, clicking inside the graph or on another "more" button causes the open pop-up to close.

If scaling is turned off, the graph will display as follows:


const greeAhnentafel2 = new PedigreeTree("ahnentafel2")
const demoAhnentafel2 = document.getElementById('demoAhnentafel2');
greeAhnentafel2.setGraph({height: "520px", width:"983px", scaling: false, clickdrag: true, multiOpen: true})
greeAhnentafel2.generate(demoAhnentafel2, ahnentafel);

Note the way that the cards are now full-sized, while users can click and drag to move the graph around within the window.
"multiOpen" is set to true to allow for the clicking and dragging operation to not close the additional details pop-up.

A family tree on a Star Wars Fan Site

A more complex usage of GreeJS could be on a Wiki page, such as the Star Wars Wiki fan page. 


const greeStarWars = new PedigreeTree("starwars");
greeStarWars.setCard({width:305, hover:true});
const demoStarWars = document.getElementById('demoStarWars');
greeStarWars.generate(demoStarWars, starwars);

Here, the card has been modified to have a width of 305px to fit the "glance" card information.
We can see that the graph contents have been scaled to fit the width of the container.
Also note that additional details now display on mouse hover over the person card, and hide when the mouse is moved away.
Pay particular attention to "Anakin Skywalker" and "Padme Amidala". These tree objects have been fully populated, and we can see that clicking on their names will redirect the user to an external wiki page. Their additional details popup has also been fully populated, with the "description" property being at the top, and the rest being placed in table format below.

The tree object is as follows:

The British Royal family tree

Another common usage for family tree would be for tracking royal blood.


const greeRoyal = new PedigreeTree("royal");
greeRoyal.setGraph({height: "520px", clickdrag:true, scalingType:"x"});
greeRoyal.setCard({cardcolor:"#E6E6FA", fontsize: 18});
const demoRoyal= document.getElementById('demoRoyal');
greeRoyal.generate(demoRoyal, royal);

Here, the height of the graph has been limited to 520px. However, the developer has chosen to scale the graph to fit the x-axis (width of the container). We can see that the graph is scrollable on the y-axis, allowing for a more aesthetic user experience without filling the whole window with a large graph.
The cards have also been modified to have a different background colour, which may fit the website's theme better, and the font-size has been increased to 18px to increase legibility.

The tree object is as follows:
